Accessible, Mobile First, Responsive Designs.

Accessible Design

I believe that making the web accessible for as many people as possible is one of the most important goals for any web developer. It not only creates better access for crucial services such as online banking, it's helping to pave the way for a better web experience for all users by encouraging a more consistent and predictable structure and behaviour for our web services.

It has never been more important that everyone of us, no matter our individual circumstances, should be able to enjoy the variety of cutting edge services that the web has to offer. A few simple attributes and some mindful planning are all it takes to ensure that your online business is accessible to as many potential users as possible.

How can more users for your online business be a bad thing? Just think of the possibilities of where this technology could lead us when we have well structured content that can be accessed cleanly and efficiently with the least amount of effort possible by the user. Imagine being able to verbally ask your computer to give you the weather forecast or a traffic report without relying on Alexa to choose which source is best for you, or having to listen to a 24 hour forecast when all you care about is the next 8 hours.

Mobile First, Responsive Architecture

It's no well kept secret that at least half of web browsing done today is coming from mobile browsers, and within certain key demographics the percentage is markedly higher. Fortunately for us web developers it has never been easier to accommodate multiple different screen sizes without resorting to browser sniffing and other scripted techniques.

By embracing a mobile first design approach, I ensure that all of my website layouts are guaranteed to look good on a user's mobile device whether they're traversing a catalog of products or navigating from page to page. By leveraging the power and flexibility of @media rules, it's simple to respond to individual screen resolutions in order to transform your content into the most appropriate layout.

Animated CSS

Recently, the HTML <canvas> element and its related APIs are replacing the need for third-party services like Adobe Flash, while CSS transitions and animations are making it simpler than ever to bring UI components to life without bloating your source code with unnecessary JavaScript. Not only does this give the browser to ability to optimize your animations by running efficiency code in the background, it also means that your website is less dependant on third party JavaScript libraries and all around easier to maintain.

By combining these different approaches, along with modern image container formats like svg and webp, it's never been simpler to bring a webpage to life while keeping page load times to a minimal.

Modern, Minimal JavaScript

There has never been better tooling in the web development world for creating compelling, captivating user experiences. From front end libraries such as React, to full featured frameworks like Angular, the only limit to what you can create is your own imagination. However, there is a time and a place for everything and many tasks involved in creating the front end experience simply don't require loading in large libraries or setting up scaffolding code in a framework just to manage a simple component or two.

In a rapidly changing industry such as web development, the knowledge of how to get things done within a certain library or framework can quickly become deprecated when the technology itself falls out of favour. With a strong knowledge of the underlying fundamentals of modern JavaScript, not only am I able to learn how to implement new technologies with minimal effort, but I am also able to design and build whole applications using only the built in objects and syntax native to virtually all browser environments as well as the emerging Node.js infrastructure.

Not only does this benefit page load times by significantly reducing the amount of third party dependencies, it also provides a greater amount of code flexibility and customization and simplifies maintenance.

On the server side, Node is making a large impact with its ever expanding collection of built in modules and an impressive selection of packages available through NPM's best-in-class dependency manager. With Node supporting the same set of ES6 features found in Chrome's V8 engine, it provides a familiar development environment with full back end capabilities.
