Accessible, Mobile First, Responsive Designs.

Accessible Design

I believe that making the web accessible for as many people as possible is one of the most important goals for any web developer. It not only creates better access for crucial services such as online banking, it's helping to pave the way for a better web experience for all users by encouraging a more consistent and predictable structure and behaviour for our web services.

It has never been more important that everyone of us, no matter our individual circumstances, should be able to enjoy the variety of cutting edge services that the modern Internet has to offer. A few simple attributes and some mindful planning are all it takes to ensure that your online business is accessible to as many potential users as possible.

Take for example a gallery of thumbnails and descriptions pertaining to a catalog of products for an online store. A popular technique would be to have the image thumbnail expand when the user passes their mouse cursor over top of it, but what if your user is taking advantage of an assisted technology that uses key presses to navigate the structure of your website instead? With accessibility in mind, we know that it only requires two additional event handlers coded in JavaScript to ensure that our AT user now has the exact same experience as someone using a more traditional mouse device.

Now, how can more users for your online business be a bad thing? And as if that isn't enough to encourage more widespread use of accessibility options, just think of the possibilities of where this technology could lead us, when we have a web full of well structured content that can be accessed cleanly and efficiently with the least amount of effort possible by the user. Imagine being able to verbally ask your computer to give you the weather forecast or a traffic report without relying on Alexa to choose which source is best for you, or having to listen to a 24 hour forecast when all you care about is the next 8 hours.

Mobile First, Responsive Architecture

It's no well kept secret that at least half of web browsing done today is coming from mobile browsers, and within certain key demographics the percentage is markedly higher. Fortunately for us web designers it has never been easier to accommodate multiple different screen sizes without resorting to browser sniffing and other relics of the not so distant past.

By embracing a mobile first design approach I ensure that all of my website layouts are guaranteed to look good on a user's phone whether they're browsing a column of images or navigating from page to page. By leveraging the power and flexibility of @media rules, it's simple to respond to larger screen resolutions in order to turn that column into a tabular layout and that expandable hamburger menu into a spacious, horizontal navigation menu.

Animated CSS

What would the web be like without a little splash of animation? From the earliest days of the web people were anxious to use animations to liven up their pages. Beginning with the gif image format created in the late 80's and notorious early experiments such as the HTML <marquee> element, developers strove to bring cutting edge animations to the Internet.

More recently, the HTML <canvas> element and its related APIs are replacing the need for third-party services like Adobe Flash while CSS transitions and animations are making it simpler than ever to bring UI components to life without writing a ton of JavaScript. Not only does this give the browser to ability to optimize your animations by running efficiency code in the background, it also means that your website is less dependant on third party JavaScript libraries and all around easier to maintain.

Modern, Minimal JavaScript

There has never been better tooling in the web development world for creating compelling, captivating user experiences. From libraries such as jQuery and React, to fully fledged frameworks like Angular and Vue, the only limit in what you can create is your own imagination. However, there is a time and a place for everything and many tasks involved in creating the front end experience simply don't require loading in large libraries or setting up scaffolding code in a framework just to manage a simple component or two.

In a rapidly changing industry such as web development, the knowledge of how to get things done within a certain library or framework can quickly become deprecated when the technology itself falls out of favour. With a strong knowledge of the underlying fundamentals of modern JavaScript not only am I able to learn how to implement new technologies with minimal study, but I am also able to design and build whole applications using only the built in objects and syntax native to virtually all browser environments as well as the emerging Node.js infrastructure.

Most modern web browsers contain a broad selection of built in APIs that make it simple to accomplish complex DOM manipulation, audio/video processing and a variety of other common and obscure use cases while third-party APIs exist for advanced services such as Google Maps and Twitter.

On the server side, Node is making a large impact with its ever expanding collection of built in modules and an impressive selection of packages available through NPM's best-in-class dependency manager. With Node supporting the same set of ES6 features found in Chrome's V8 engine, my knowledge of modern JavaScript allows me to write anything from a simple web scraping script to a fully fledge web application while being able to incorporate modules into my code as needed.
